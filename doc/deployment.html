Deployment

CI/CD

  • Garnix runs nix flake check on every push and PR (tests, clippy, formatting)
  • GitHub Actions deploys to Cloudflare Workers on push to main
  • GitHub Actions builds and deploys documentation to GitHub Pages on changes to docs/

Manual Deploy

nix develop
wrangler deploy

Database Migration

Apply the schema migration (creates all tables, safe to re-run with IF NOT EXISTS):

wrangler d1 execute concierge-worker --remote --file migrations/0001_create_schema.sql

This creates the following tables:

TablePurpose
tenantsTenant accounts
whatsapp_messagesWhatsApp message metadata log
instagram_messagesInstagram message metadata log
lead_form_submissionsLead form submission records
email_messagesEmail routing action log
email_metricsAggregated email routing counters by domain, rule, date, and action
messagesUnified message metadata log (all channels, no content stored)
credit_packsAvailable credit pack definitions (managed via management panel)
paymentsRazorpay payment history
audit_logManagement action audit trail

The migration also seeds four default credit packs (Starter, Growth, Scale, Volume).

To reset the database (drops all data):

wrangler d1 execute concierge-worker --remote --command "DROP TABLE IF EXISTS tenants; DROP TABLE IF EXISTS whatsapp_messages; DROP TABLE IF EXISTS instagram_messages; DROP TABLE IF EXISTS lead_form_submissions; DROP TABLE IF EXISTS email_messages; DROP TABLE IF EXISTS email_metrics; DROP TABLE IF EXISTS messages; DROP TABLE IF EXISTS credit_packs; DROP TABLE IF EXISTS payments; DROP TABLE IF EXISTS audit_log;"
wrangler d1 execute concierge-worker --remote --file migrations/0001_create_schema.sql

Razorpay Webhook Setup

  1. In the Razorpay Dashboard, go to Settings > Webhooks
  2. Add a new webhook URL: https://your-domain/webhook/razorpay
  3. Select events: payment.captured, payment.failed
  4. Copy the webhook secret and set it: wrangler secret put RAZORPAY_WEBHOOK_SECRET

Cloudflare Access Setup for /manage/*

  1. In the Cloudflare dashboard, go to Zero Trust > Access > Applications
  2. Create a Self-hosted application
  3. Set the application domain to your-domain and path to /manage/*
  4. Copy the Application Audience (AUD) Tag and set it as CF_ACCESS_AUD in wrangler.toml. Set CF_ACCESS_TEAM to your team name.
  5. Create a policy allowing the emails that should have management access
  6. Concierge verifies the Cf-Access-Jwt-Assertion JWT signature and audience | no spoofable headers

Email Routing Setup

  1. In the Cloudflare dashboard, go to Email Routing for your domain
  2. Enable Email Routing and verify the domain
  3. Add a Catch-all rule that routes to your Worker (concierge-worker)
  4. The Worker's email() handler receives all inbound emails for the domain

Pre-commit Hooks

The Nix dev shell installs git hooks via cachix/git-hooks.nix:

  • rustfmt: code formatting
  • nixpkgs-fmt: Nix formatting
  • check-toml, check-yaml, check-json: config validation
  • detect-private-keys: prevents accidental key commits
  • end-of-file-fixer, trim-trailing-whitespace: file hygiene
