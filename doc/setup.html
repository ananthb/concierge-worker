Setup Guide

Prerequisites

  • A Cloudflare account with Workers, D1, KV, and Email Routing enabled
  • A Meta Developer account with a Facebook App
  • A Google Cloud project (for OAuth sign-in)
  • A Discord application (for the relay bot and slash commands)
  • A Razorpay account (for paid credit packs, optional)
  • Nix (for the development environment)

1. Clone and Build

git clone https://github.com/ananthb/concierge-worker
cd concierge-worker
nix develop
cargo test

2. Create Cloudflare Resources

# Create D1 database
wrangler d1 create concierge-worker

# Create KV namespace
wrangler kv namespace create KV

Update the IDs in wrangler.toml.

3. Apply Database Migration

wrangler d1 execute concierge-worker --remote --file migrations/0001_create_schema.sql

This creates all tables: tenants, whatsapp_messages, instagram_messages, lead_form_submissions, email_messages, email_metrics, messages, credit_packs, payments, and audit_log. It also seeds the default credit packs.

4. Set Secrets

See Configuration & Secrets for the full list. At minimum you need:

# Core encryption
wrangler secret put ENCRYPTION_KEY

# Google OAuth
wrangler secret put GOOGLE_OAUTH_CLIENT_ID
wrangler secret put GOOGLE_OAUTH_CLIENT_SECRET

# Meta (Facebook/Instagram/WhatsApp)
wrangler secret put META_APP_ID
wrangler secret put META_APP_SECRET
wrangler secret put WHATSAPP_ACCESS_TOKEN
wrangler secret put WHATSAPP_VERIFY_TOKEN
wrangler secret put INSTAGRAM_VERIFY_TOKEN

# Discord
wrangler secret put DISCORD_PUBLIC_KEY
wrangler secret put DISCORD_APPLICATION_ID
wrangler secret put DISCORD_BOT_TOKEN

# Razorpay (optional, for paid billing)
wrangler secret put RAZORPAY_KEY_ID
wrangler secret put RAZORPAY_KEY_SECRET
wrangler secret put RAZORPAY_WEBHOOK_SECRET

5. Configure wrangler.toml Variables

Update these in the [vars] section:

  • CF_ACCESS_TEAM: Your Cloudflare Access team name (e.g. myteam)
  • CF_ACCESS_AUD: Application Audience (AUD) tag from your Access application for /manage/*
  • WHATSAPP_WABA_ID: Your WhatsApp Business Account ID
  • WHATSAPP_SIGNUP_CONFIG_ID: Meta Embedded Signup configuration ID

6. Deploy

wrangler deploy

7. Configure Webhooks

After deploying, configure webhooks in the Meta Developer Console:

  • WhatsApp: Callback URL https://your-domain/webhook/whatsapp, subscribe to messages
  • Instagram: Callback URL https://your-domain/webhook/instagram, subscribe to messages

See Facebook App Setup for detailed instructions.

8. Set Up Discord Bot

  1. Create a Discord application at discord.com/developers
  2. Set the Interactions Endpoint URL to https://your-domain/discord/interactions
  3. Create a bot user and copy the token
  4. Register slash commands (/status, /domains, /rules) via the Discord API
  5. Invite the bot to your server with appropriate permissions

See Discord Integration for details.

9. Set Up Email Routing (inbound)

The email domain cncg.email is already configured (zone ID 5cf43b684849d5805eb0a115a18dbee2). To complete setup:

  1. In the Cloudflare dashboard, go to Email Routing for cncg.email
  2. Add a catch-all routing rule: Action → Send to a Worker → select concierge-worker
  3. Set the DNS API token: wrangler secret put CF_DNS_API_TOKEN (a CF API token with DNS Edit on the cncg.email zone)

Tenant subdomains (e.g. acme.cncg.email) are provisioned automatically via the admin panel. MX records are auto-discovered from the apex domain. See Email Routing for details.

10. Set Up Email Service (outbound)

Email Routing's send_email binding alone only delivers to addresses pre-verified in Cloudflare. To forward to any recipient — which is the whole point of Concierge's email forwarding — onboard cncg.email to Cloudflare's Email Service:

  1. Cloudflare dashboard → Email → Email Sending → Onboard Domain
  2. Accept the DNS records (MX / SPF / DKIM / DMARC) Cloudflare adds on the cf-bounce.cncg.email subdomain. These are independent of any root-domain SPF/DKIM you use for transactional mail elsewhere.

Outbound email is sent via Email Service's structured API ({from, to, subject, text, html, replyTo, headers}), not raw MIME bytes. Forwarded mail keeps subject, text/html bodies, and threading headers; attachments and exotic multipart structures are not preserved.

Razorpay Webhook (for subscription billing)

Email subdomain billing plans are created automatically via the Razorpay API — no manual plan setup needed. Just configure the webhook:

  1. In the Razorpay Dashboard, go to Settings > Webhooks
  2. Add webhook URL: https://your-domain/webhook/razorpay
  3. Subscribe to events: payment.captured, payment.failed, subscription.activated, subscription.charged, subscription.halted, subscription.cancelled
  4. Copy the webhook secret and set it: wrangler secret put RAZORPAY_WEBHOOK_SECRET

11. Set Up Cloudflare Access for Management

  1. In the Cloudflare dashboard, go to Zero Trust > Access > Applications
  2. Create an application for your-domain/manage/*
  3. Copy the Application Audience (AUD) Tag and set it as CF_ACCESS_AUD in wrangler.toml
  4. Add a policy allowing the emails that should have management access

See Management Panel for details.

12. Set Up Razorpay (Optional)

  1. Create a Razorpay account and get your API keys
  2. Set up a webhook in Razorpay pointing to https://your-domain/webhook/razorpay
  3. Set the webhook secret and API keys as Worker secrets

See Billing for details.
